差分隐私学习总结

发表于 2022-08-04 更新于 2022-08-08 分类于 机器学习 ， 隐私保护
# 一、思维导图 ![](https://cdn.nlark.com/yuque/0/2022/jpeg/27569747/1659886767738-0d766194-b759-484d-9e74-b6e88cf34324.jpeg) # 二、读书笔记 ## 2.1 前言部分

  1. 信息技术的发展，对数据集的研究成为了各个领域的关键，数据成为了最重要和基础的资源，数据的发布、共享、分析成为了一个热门的话题

  2. 数据集中往往包含了诸多的隐私，隐私会随着数据的发布而被泄露。保护隐私，就是要使得任何用户访问数据库的过程中无法获取关于任何个体的确切信息。

  3. “K-匿名”方法被提出，通过分组的方式，在进行压缩、泛化等一系列处理后，将所有记录划分到若干等价类中，从而将单个记录隐藏在一整个类中。

  4. “K-匿名”方法存在两个问题：①攻击者所掌握的背景知识难以定义，总是因为出现新型的攻击方式而不断更新；②无法对隐私保护水平进行定量分析，从而也就无法用严格的数据来评估保护方法的能力。

  5. “差分隐私”方法被提出，相比“K-匿名”方法，存在两个特点：①假设攻击者能够掌握除了攻击目标之外的所有背景知识，默认以最恶劣的环境来处理问题；②对隐私保护进行了严格的定义，并提供了量化评估的方法。

    2.2.3 隐私保护预算

    我们使用ε来表示隐私保护预算，ε越小，保护的水平越高，但相对的，结果也不能反映出任何有关数据集的有用信息了。安全性和可用性不可兼得。
    根据之前的推导，我们是使用“信息熵”来表示差分隐私的，因此ε可以理解为为了达到尽可能高的安全性而使得信息可用性降低到的水准。

相对熵衡量两个分布之间的差别，而两个分布对于任一事件的差别都被ε限制住了，ε越小，相对熵越小，自然分布差别越小，安全性越高，可用性越小。
可用性的降低还可以这样理解：为了使得隐私保护提高，必须加强噪声，而噪声会使得数据的可用性能降低。这样，“预算”的概念又可以变为“可以施加的噪声量”，每次对数据进行访问，就在预算中进行噪声添加，预算削减。而当预算变为0时，就没法进行有价值的分析了。
原理上来讲，上述两者的分析是等价的。

2.2.4 差分隐私保护算法的组合性质

性质1 序列组合性
设有算法M1，M2，……，Mn，其隐私保护预算分别为ε1，ε2，……，εn，那么对于同一数据集D，由这些算法构成的组合算法M(M1(D),M2(D),……,Mn(D))，提供（Σεi）-差分隐私保护。
这种组合算法，预算为全体预算的总和。
性质2 并行组合性
设有算法M1，M2，……，Mn，其隐私保护预算分别为ε1，ε2，……，εn，那么对于不相交的数据集D1，D2……Dn，由这些算法构成的组合算法M(M1(D1),M2(D2),……,Mn(Dn))，提供（max ε）-差分隐私保护。
如果算法序列中，所有算法处理的数据集彼此不相交，那么预算为所有预算中最大值，也即隐私保护水平取决于算法序列中保护水平最差者。

【推导】
首先我们要明白，指数机制的含义是什么。在指数机制的定义中，我们引入了一个“可用性函数”，也可以将其称之为“utility function”，即“效用函数”。它定义了一种对于数据的查询方式。
这种查询方式是有问题的，例如通过最大化效用函数与其他信息的组合，我们就可以推断出某个人的隐私。因此，反过来，如果让每个数据在算法M的处理之后，输出结果为效用得分更高的那种选择的话，暴露的隐私就尽可能地少了。这就是指数机制的含义：以更可能高的概率，得到效用得分更高的输出。

2.2.6 研究方向

集中于“数据发布”和“数据挖掘”中。

方向 类型 方式
数据发布 交互式 用户提出查询请求，管理员对数据进行扰动处理后返回给用户，用户无法看到数据集全貌
非交互式 管理员对所有数据进行处理，在数据符合差分隐私的前提后，一次性全部发布
数据挖掘 接口模式 用户不可信，管理员只对外发布接口，并对接口进行差分隐私保护，查询数目受隐私保护预算限制
完全访问模式 对数据查询量没有限制，要求用户自己设计查询算法保证差分隐私，确保发布的模型不会泄露数据

这个不等式的意义在于：查询结果和干扰结果的差越小，精确度越高。当达到该精确度的概率大于某个确信值时，我们认为这次查询是有意义的。

2.3.5 批查询

【定义】管理者针对所有可能的查询，一次性对外发布所有查询的结果

方法 实现 总敏感度 噪声方差
Lap1 每个范围查询加入独立噪声 O(n^2) O(n^4/ε^2)
Lap2 先为每个单一查询加入噪声，再根据干扰后的单一查询计算其他范围查询结果 O(1) O(n/ε^2)
小波变换 将频数统计矩阵映射到小波系数矩阵中，再为每个小波系数添加Laplace噪声 O(logn+1) O((logn)^3d/ε^2)
层次查询 基于层次树，将频数统计矩阵映射到层次树中，再添加噪声 O(logn+1) O((logn)^3/ε^2)

2.3.6 列联表发布方法

【定义】假设数据集D包含n个记录，由K个布尔变量组成，那么对数据集按照 2^k个可能的组合值进行统计记数所形成的表格，就是列联表。按照多个变量的组合值进行统计所得的记数，也称为“边缘频数”。

方法 实现 特点
Lap1 向列联表的每个单元格加入噪声，根据干扰频数计算边缘频数 保持了一致性，但累计噪声偏大
Lap2 先计算拟发布的边缘频数，再加入噪声 可用性好，可能违背一致性
Barak 将列联表的数据进行傅里叶转换，对边际频数进行无冗余编码 保持了一致性，噪声小，但不适用于稀疏数据集和非布尔变量数据集

2.3.7 基于分组的数据发布方法

【定义】延续了“K-匿名”模型的思想，并用差分隐私的要求来控制分组和匿名处理的整个过程。

方法 实现 特点
“安全-K-匿名”模型 1、确定一个随机抽样函数Am，对数据集D进行抽样，得到Sd=Am(D)
2、从Sd中删除出现次数小于k的记录		 1、当函数Am满足ε-差分隐私保护条件时，“安全-k-匿名”模型能够提供(ε，δ)-差分隐私保护
2、只是个理论框架
DiffGen发布算法 1、对每个连续属性值的细分方案，应用指数机制从Nj中选择一种方案，得到y2种方案
2、分别与y1种离散属性组合，采用指数机制从这y1+y2种方案中选择一种
3、不断迭代循环，直到达到预定的细分次数		 没有充分利用给定的预算，加入了不必要的冗余噪声
DiffGen-改 每次细分迭代种。将所有连续属性细分方案乘以相应的权重后，和离散属性细分方案一起构成候选方案集，再调用指数机制进行选择 减少了调用指数机制的次数，提高了预算的利用率

2.3.8 净化数据集发布方法

【定义】将学习理论引入到隐私保护当中来，当数据集的样本复杂度达到O(VC(C)/a^2)时，对于任意查询f∈C，D中正例的比例与X中正例的比例误差在±a之内。

方法 实现 特点
差分隐私PAC定义 可学习：
Pr(err(h)≤a)≥1-δ
不可知学习：
Pr(err(h)≤OPT+a)≥1-δ		 如果一个概念类别在无隐私保护要求和多项式样本复杂度下是可学习的，那么它在差分隐私保护条件下也是可学习的。
Blum等人的研究 略 对于任意具有多项式VC维度的概念类别，不考虑计算复杂度，采用指数机制遍历整个数据域，最终找到满足差分隐私保护的数据集是可行的。

2.4 基于差分隐私保护的数据挖掘

2.4.1 两种广泛应用的接口框架

  1. SuLQ框架：首先将单属性布尔查询定义为查询原语，然后以原语为基本操作，组合查询集合，实现更复杂的查询函数。将数据类型扩展到连续型数据，对原语设计提供隐私保护功能的复杂算法。
  2. PINQ框架：基于LINQ查询语言。由Partition算子将数据集分割成不相交的子集，然后利用差分隐私保护算法的并行组合性，提高隐私保护预算的利用率。

    2.4.2 接口模式下的分类算法

    【目标】：根据训练数据集建立分类器模型，用以推测新纪录的类标识。

    2.4.3 接口模式下的聚类算法

    【目标】将无类标识的记录划分到若干个簇中，使得簇内记录相似度高，簇间相似度低。此外，从数据集中删除任一记录时，要能保证每个聚类的质心以及记录数量所发生的改变不泄露隐私信息。
方法 实现 特点
SuLQ框架下的k-means算法 通过发布聚类质心和记录数据量的估计值来满足隐私保护的要求 查询聚类质心的函数敏感度为聚类的最大直径，以此计算的噪声量偏大
抽样—聚合框架 1、从原始数据集D中随机抽样产生m个子集，分别带入函数f得到m个中间结果
2、然后选择一个局部敏感度低并容易计算的聚合函数g，对m个中间结果进行聚合运算，得到一个关于f(D)的期望值
3、再根据g的平滑敏感度向该期望值中添加噪声。		 对所有随机抽样生成的数据集进行聚类且输出的结果具备某种程度上的一致性是，算法才可能输出令人满意的结果
基于核心集的方法 给定一个d维空间里的点集G，样本容量为n，可以计算出一个带权重且容量为O(ka^(-d)logn)的点集Sg，用Sg代替G做聚类分析，得到一个近似结果。 略
基于预算分配的k-means差分隐私保护完善算法 若算法迭代次数指定为u，则每次迭代的预算为ε/u，添加的噪声服从Lap((d+1)u/ε)分布；
若算法迭代次数不定，则第一次分配的预算为ε/2，后面每次依次为前者的1/2		 略

2.4.4 完全访问模式下的分类算法

方法 实现 特点
随机决策树 1、去掉了构建模型过程中的剪纸步骤
2、取出全部叶节点中所有可能的类标签计数值，形成向量V
3、向V中加入Lap(1/ε)噪声		 如果分类器中包含k棵树，那么构建每棵树所分配的预算为ε/k
logistic回归算法 先将噪声加在目标函数的参数中，然后通过标准的logistic回归算法进行参数估计来获取模型 相比直接利用拉普拉斯机制，具有更高的分类准确率

2.4.5 完全访问模式下的频繁项集挖掘

方法 实现 特点
截断支持度 1、利用传统挖掘算法，计算出格子的截断支持度s^
2、指数机制的FIM算法以s^为可用性函数K次调用指数机制；Laplace机制的FIM算法对预处理的每个s^添加噪声，再选支持度最大的K个项集
3、对选定的K个项集，为它们的真实支持度添加Laplace噪声，输出		 1、可以提供ε-差分隐私。以1-ρ的概率实现两个可用性要求
2、执行前必须预定义输出项集的长度l，不能根据中间结果自适应 l的值
3、K较小时结果较好，K较大时，性能和准确性会显著下降
PrivBasis 希望能够找到一个项集IB，使得最频繁的K个项集均为IB的子集，然后通过计算并利用IB中1-项集和2-项集的支持度，重构IB中所有子集的支持度，并添加相应的噪声 1、可以避免遍历所有可能的项集，降低计算代价
2、K很大时，IB必将很大，不能从根本上解决问题
θ-Basis Set 任何一个支持度大于θ的项集都是某个IBi的子集 在频繁项集漏报率和支持度误差两个方面都低于截断支持度方法
